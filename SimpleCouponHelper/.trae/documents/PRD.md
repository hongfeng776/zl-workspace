## 1. 产品概述

对歌系统是一款在线K歌对战平台，支持双人实时对唱、自动评分和音频录制功能。用户可以注册登录、邀请好友、点歌对唱，系统通过音频分析自动评分判定输赢，并支持保存演唱音频供后续回放。

- **主要目的**：提供有趣的在线K歌对战体验，增加社交互动性
- **目标用户**：喜欢唱歌、音乐社交的年轻用户群体
- **产品价值**：打破传统单人K歌模式，通过对战机制增加趣味性和挑战性

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 普通用户 | 手机号/用户名注册 | 注册登录、创建房间、加入房间、点歌、对唱、查看评分、保存音频 |

### 2.2 功能模块

1. **用户认证模块**：注册、登录、个人信息管理
2. **歌房大厅模块**：房间列表、创建房间、加入房间、邀请好友
3. **点歌系统模块**：歌曲库、歌曲搜索、点歌、歌曲详情
4. **对唱对战模块**：歌词同步显示、录音、实时音频分析、自动评分
5. **评分结果模块**：得分展示、输赢判定、历史记录
6. **音频管理模块**：音频保存、回放、下载

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 登录注册页 | 用户认证 | 用户名密码登录、新用户注册、表单验证 |
| 歌房大厅页 | 大厅模块 | 房间列表展示、创建房间、快速加入、邀请好友弹窗 |
| 歌房房间页 | 对唱模块 | 房间信息、成员状态、点歌面板、歌词展示区、录音控制 |
| 歌曲库页 | 点歌模块 | 歌曲列表、分类筛选、搜索功能、点歌操作 |
| 评分结果页 | 评分模块 | 双方得分对比、详细评分项、输赢判定、保存音频 |
| 个人中心页 | 用户模块 | 个人信息、历史记录、我的音频、设置 |

## 3. 核心流程

用户注册登录后，可创建或加入歌房，邀请好友进入同一房间。双方准备就绪后开始点歌，选择歌曲后进入对唱模式。系统同步显示歌词并录制双方音频，通过音频分析算法（音准、节奏、完整性等维度）进行实时评分。对唱结束后展示评分结果，判定输赢并保存音频文件。

```mermaid

    A["用户注册/登录"] --> B["进入大厅"]
    B --> C["创建/加入房间"]
    C --> D["邀请好友"]
    D --> E["点歌选择"]
    E --> F["开始对唱"]
    F --> G["实时录音&评分"]
    G --> H["展示结果"]
    H --> I["保存音频"]
    I --> J["返回大厅/继续对战"]
```

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：深紫色(#6366F1) 搭配 霓虹粉色(#EC4899)，营造音乐派对氛围
- **辅助色**：青色(#06B6D4) 用于强调交互元素
- **背景**：深色渐变背景，配合音乐波形动效
- **按钮风格**：圆角胶囊型按钮，带有霓虹发光效果，悬停时有缩放动画
- **字体**：展示字体使用 Orbitron（科技感），正文字体使用 Noto Sans SC
- **布局风格**：卡片式布局，玻璃拟态效果，毛玻璃背景
- **图标风格**：线性图标，配合霓虹发光效果
- **动效**：音乐波形动画、粒子效果、玻璃拟态

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 登录注册页 | 用户认证 | 玻璃拟态卡片、渐变按钮、动态背景波形、输入框发光效果 |
| 歌房大厅页 | 大厅模块 | 房间卡片网格、创建房间悬浮效果、房间状态指示灯 |
| 歌房房间页 | 对唱模块 | 左右分屏对战布局、歌词滚动区、波形可视化、录音按钮 |
| 歌曲库页 | 点歌模块 | 歌曲列表卡片、播放预览、搜索框发光效果 |
| 评分结果页 | 评分模块 | 雷达图评分对比、分数动画、胜负徽章、彩带特效 |
| 个人中心页 | 用户模块 | 头像卡片、历史记录时间线、音频列表 |

### 4.3 响应式

- 采用桌面端优先设计，适配1024px以上屏幕
- 平板端自适应布局，歌房页面左右分屏改为上下布局
- 移动端优化触摸交互，调整卡片尺寸

### 4.4 动效设计

- 页面加载时的渐入动画和元素交错显示
- 录音时的波形可视化动效
- 评分时的数字滚动动画
- 按钮悬停时的霓虹发光效果
- 歌词滚动时的高亮和当前行放大效果
